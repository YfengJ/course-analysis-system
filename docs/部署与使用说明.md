# 课程目标达成度报告系统部署与使用说明

本文档用于说明系统的本地部署、初始化、日常使用、数据备份和发布包更新方式。

## 1. 运行环境

- Python 3.10 或以上版本，推荐 Python 3.12
- 可访问本机浏览器
- SQLite 随 Python 自动可用，无需单独安装数据库
- 如需使用智能建议功能，需要准备可用的模型服务地址和密钥；未配置密钥时，系统的建课、导入、计算、报告导出和备份恢复仍可正常使用

## 2. 解压发布包

将发布包解压到一个固定目录，例如：

```text
/Users/用户名/course-analysis-system
```

目录中应包含：

```text
app.py
config.py
init_db.py
requirements.txt
routes/
services/
templates/
static/
sample_data/
docs/
```

发布包不包含历史数据库、上传文件、导出报告、真实课程资料、模型密钥和本地缓存。

## 3. 创建运行环境

在项目目录中执行：

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python init_db.py
python app.py
```

Windows PowerShell 可使用：

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
python init_db.py
python app.py
```

启动后访问：

```text
http://127.0.0.1:5000
```

首次启动会自动创建管理员账号：

```text
账号：admin
初始密码：admin123
```

首次登录后系统会要求修改初始密码，完成后才能进入课程数据页面。

## 4. 推荐配置

建议把数据库、上传文件、导出报告和备份文件放到源码目录之外，便于升级和备份。可以在启动前设置：

```bash
export SECRET_KEY="请替换为本机随机字符串"
export COURSE_SYSTEM_DATA_DIR="/Users/用户名/course-system-data"
export DEFAULT_ADMIN_USERNAME="admin"
export DEFAULT_ADMIN_PASSWORD="请替换为临时强密码"
```

如需启用智能建议功能，再配置：

```bash
export LLM_API_BASE="https://api.deepseek.com"
export LLM_API_KEY="请填写自己的模型服务密钥"
export LLM_MODEL="deepseek-v4-flash"
export LLM_TIMEOUT="45"
```

`LLM_TIMEOUT` 的单位是秒。不要把 `.env`、密钥、数据库、真实成绩表或导出报告放入发布包。

## 5. 基本使用流程

1. 登录系统并修改初始密码。
2. 在“开始建课”中新建课程，或上传教学大纲自动建课。
3. 进入课程概览页，检查课程名称、课程编号、院系、专业、课程负责人、任课人员、学期等基础信息。
4. 在教学大纲页确认课程目标、毕业要求指标点、考核项和权重是否正确。
5. 在成绩导入页上传成绩文件。可一次选择多个文件，用于合并多个班级成绩。
6. 先执行预检，确认学生数、班级、工作表、列映射和分值异常提示。
7. 预检无误后确认导入。
8. 进入计算分析页，执行达成度计算；如有必要，可修订定性评价计数和说明。
9. 进入第五章编辑页，生成或手工填写评价结果分析与持续改进措施。
10. 在报告预览页查看报告内容，运行报告质量检查。
11. 质量检查通过后导出 Word 报告。
12. 报告确认无误后可归档为最终版，并导出课程归档包。

## 6. 报告质量检查

报告质量检查会检查以下内容：

- 课程负责人是否填写
- 课程目标是否仍为占位描述
- 课程目标与毕业要求指标点是否建立映射
- 考核项是否绑定课程目标权重
- 当前学期和班级范围是否已有成绩
- 最近一次导入是否存在异常
- 第四章计算是否完成
- 第五章内容是否保存
- 最新报告是否已归档

普通检查用于日常提醒；归档最终版时会启用严格检查，关键问题会阻止归档。

## 7. 数据备份与恢复

在“数据维护”页面可以创建备份包。备份包包含：

- SQLite 数据库
- 上传过的教学大纲和成绩文件
- 已生成的报告文件
- 备份清单

恢复备份前需要输入：

```text
确认恢复
```

系统会先自动创建一份当前数据的完整备份，再执行恢复。恢复包必须由当前系统创建，并包含版本匹配的 `manifest.json`。

建议在以下场景创建备份：

- 每次集中导入成绩前
- 每次批量生成报告后
- 每次升级系统前
- 学期报告归档完成后

## 8. 发布包更新

如需重新生成不含真实数据的发布包，在项目目录执行：

```bash
python scripts/build_release.py
```

默认输出：

```text
dist/course-system-release.zip
```

发布脚本会自动排除：

- `.env`、模型密钥和本地配置
- 数据库和备份文件
- `uploads/`、`exports/`、`instance/`、`tmp/`、`output/`
- 真实成绩表、教学大纲和导出报告
- 本地虚拟环境、浏览器工具、IDE 配置、缓存和过程文档

## 9. 常见问题

### 9.1 忘记修改后的密码怎么办

如果只有本地单机使用，可先备份数据，再联系维护人员重置数据库中的用户密码。不要直接删除数据库，避免课程和报告记录丢失。

### 9.2 没有配置模型密钥能不能用

可以。未配置模型密钥时，只影响智能建议生成；课程管理、教学大纲导入、成绩导入、达成度计算、报告导出、备份恢复都可以使用。

### 9.3 成绩文件导入后结果不对怎么办

先查看成绩导入页的预检结果和导入日志，重点检查：

- 是否选择了正确的工作表
- 学号、姓名、班级列是否识别正确
- 考核项列名是否与教学大纲中的考核项一致
- 分值是否超过该考核项满分
- 多个班级文件是否重复导入

必要时可删除本次课程成绩后重新导入；操作前建议先创建系统备份。

### 9.4 报告无法归档怎么办

进入报告质量检查页查看阻断项。常见原因包括课程目标仍是占位描述、毕业要求映射缺失、第四章未计算、第五章未保存或最近一次导入存在异常。

## 10. 维护建议

- 定期创建备份包，并保存在项目目录之外。
- 不要把真实课程资料、学生成绩、数据库、导出报告和模型密钥放入源码仓库。
- 每次升级前先备份，再替换系统源码。
- 发布包只用于安装和升级；运行产生的数据应保存在独立数据目录中。
